Excel Estimation Add-in — Local Setup

This guide explains prerequisites and how to run the project locally on Windows with Excel Desktop.

1) Prerequisites
- Windows 10/11
- Microsoft Excel (Desktop) with a Microsoft 365 account
- Node.js (LTS). Verify:
  - node -v
  - npm -v
- Python 3.10+ available on PATH (py, python, or python3). The dev server will try these in order.
- SQLite database files present in the project root:
  - my_database.db (for Mhr lookups; table name: my_table, must include columns: description, "Cable laying Mhr", "Cable termination Mhr")
  - my2.db (for Price lookups; default table name: gpt_purchase_data_(1), with columns: Description, price)
- Optional: Loopback exemption approved for WebView2 when debugging Office add-ins. If Excel shows Access Denied, run VS Code as Administrator once or approve the exemption.

2) Install dependencies
- From VS Code Terminal or PowerShell in the project folder:
  - npm install

3) Configure environment (optional)
- You can override defaults via environment variables before starting the dev server:
  - PRICE_DB: full path to an alternate price DB (default: ./my2.db)
  - PRICE_TABLE: table name for price data (default: gpt_purchase_data_(1))
  - PRICE_DESC_COL: description column name (default: Description)
  - PRICE_VALUE_COL: price column name (default: price)
  - PRICE_MAX_CANDIDATES, PRICE_MIN_SCORE, PRICE_MIN_RESULTS_BEFORE_THRESHOLD: tuning knobs for matching
- For the Mhr UI app in Python (mhrsqlsystem.py), configure at top if you run it standalone:
  - OPENAI_API_KEY, GPT_MODEL, DB_FILENAME/DB_TABLE
  Note: The add-in does not require your OpenAI key; the Python pipe uses your local DB. The code currently contains a hard-coded API key used only by the standalone Tkinter app path.

4) Start the add-in in Excel Desktop
Option A — Using VS Code tasks
- Run: Start Add-in (Excel Desktop)
  This executes: npm run start -- desktop --app excel

Option B — Using npm directly
- In the project folder run:
  - npm run start -- desktop --app excel

What happens on start
- A HTTPS dev server launches on https://localhost:3000
- Webpack devServer automatically spawns two Python helper processes:
  - mhr_pipe.py (uses mhrsqlsystem.py to resolve Mhr)
  - price_pipe.py (uses pricesqlsystem.py to resolve Price)
- The add-in is sideloaded into Excel via manifest.xml and opens the task pane.

5) Using the add-in
- In Excel: Home → Commands Group → Show Task Pane
- Slots available in the sidebar:
  - Mhr & Cost calculator: Reads Description column, computes Mhr via /api/mhr, and (optionally) Cost if a Rate column exists. It creates headers only if missing and writes into the chosen columns, preserving layout.
  - Price calculator: Reads Description, resolves unit price via /api/price into a Price column placed to the right of Description if available, preserving layout.
  - Cable Filler and Templates: Utilities for worksheet formatting and inserting a template sheet.

6) Data requirements
- For Price (my2.db): default schema expectations
  - Table: gpt_purchase_data_(1)
  - Columns: Description (text), price (numeric/text parseable)
  - You can change names via env vars listed in step 3.
- For Mhr (my_database.db): schema expectations
  - Table: my_table
  - Columns: description, "Cable laying Mhr", "Cable termination Mhr"

7) Troubleshooting
- Excel opens but add-in pane is blank:
  - Ensure https://localhost:3000 is accessible; check the Dev Server terminal for errors.
- "Python pipe not running" errors on /api/mhr or /api/price:
  - Ensure Python 3 is installed and on PATH. Set PYTHON environment variable to the full interpreter path if needed.
  - Verify mhr_pipe.py and price_pipe.py can import mhrsqlsystem.py / pricesqlsystem.py and that the DB files exist.
- Price/Mhr lookups return blanks:
  - Check that the DB files exist and contain data with matching descriptions.
  - Adjust PRICE_* environment variables for different column/table names.
- Access Denied / WebView loopback:
  - Run VS Code as Administrator once or approve loopback exemption when prompted.
- Certificate trust errors:
  - Allow and trust the dev certificate generated by office-addin-dev-certs when prompted.

8) Useful scripts and tasks
- Build (Development): webpack --mode development
- Build (Production): webpack --mode production
- Dev server only: npm run dev-server (does not sideload Excel)
- Lint: npm run lint
- Stop debug: npm run stop

9) Optional: Run Python Mhr GUI tool
- You can run the standalone Tkinter estimator app (unrelated to the add-in UI):
  - python mhrsqlsystem.py
  Configure OPENAI_API_KEY and the local DB path at the top of that file. This is optional and not required for the add-in.

Notes
- This project uses Office Add-ins Dev Kit conventions. The add-in UI and logic live under ./src. The manifest sideloads into Excel Desktop.
- During development, the add-in calls the local endpoints /api/mhr and /api/price served by the webpack dev server, which bridge to the Python pipes.
